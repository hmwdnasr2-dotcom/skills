chat Chat Workflow

  1. 1. User sends message via Web App or Telegram
  2. 2. Intent router classifies request (task / email / research / general)
  3. 3. Context builder pulls memory + relevant data
  4. 4. ARIA Core generates response using active connectors as tools
  5. 5. Memory layer saves insight for future context
  6. 6. Response delivered back to user

schedule Proactive Workflow

  1. 1. Scheduler fires on cron (morning, evening, reminders)
  2. 2. Builds context from pending tasks + reminders
  3. 3. ARIA generates proactive briefing or nudge
  4. 4. Pushed via SSE to open web sessions
  5. 5. Sent to Telegram if configured

attach_file File Processing Workflow

  1. 1. User attaches file (PDF, Excel, CSV, image, doc)
  2. 2. Upload endpoint stores file, returns fileId
  3. 3. Chat sends message + fileIds to ARIA
  4. 4. File handler extracts text / parses data
  5. 5. ARIA analyzes content with user's prompt
  6. 6. Optional: generate Excel/PDF report as output

memory Memory Workflow

  1. 1. After each ARIA response, insight is extracted
  2. 2. Saved to Supabase aria_memory table
  3. 3. On next message, recent memories loaded as context
  4. 4. If OpenAI key set: vector embeddings for semantic search
  5. 5. Search memories from chat header search bar
